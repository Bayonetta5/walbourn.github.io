<p>Windows 8 includes an updated &ldquo;DirectX 11.1 Runtime&rdquo; that supports <a href="http://msdn.microsoft.com/en-us/library/hh404562.aspx">Direct3D 11.1</a>, updates <a href="http://msdn.microsoft.com/en-us/library/windows/desktop/hh802478.aspx">Direct2D</a> and <a href="http://msdn.microsoft.com/en-us/library/windows/desktop/hh802480.aspx">DirectWrite</a>, <a href="http://msdn.microsoft.com/en-us/library/hh404490.aspx">DXGI 1.2</a>, and a revision of the <a href="http://msdn.microsoft.com/en-us/library/hh994467.aspx">Windows Imaging Component (WIC)</a>.</p>
<p>Portions of the &ldquo;DirectX 11.1 Runtime&rdquo; are being made available on <a href="http://blogs.msdn.com/b/chuckw/archive/2011/02/22/windows-7-service-pack-1.aspx">Windows 7 Service Pack 1</a> via the <em>Platform Update for Windows 7 Service Pack 1 and Windows Server 2008 R2 Service Pack 1</em> (KB 2670838) included with <em>Internet Explorer 10 for Windows 7</em>. This includes the updated components above, but is limited to WDDM 1.1 drivers on Windows 7.</p>
<p>Full technical details&nbsp;on what is&nbsp;and is not included in the update&nbsp;are available on <a href="http://msdn.microsoft.com/en-us/library/jj863687.aspx">MSDN</a>. For information about IE10 compatibility, see this <a href="http://msdn.microsoft.com/en-us/library/ie/jj819730.aspx">article</a>.</p>
<p><strong>Note:</strong> KB 2670838 does not include XINPUT 1.4 or XAudio 2.8 on Windows 7. These remain Windows 8 exclusive. See <a href="http://blogs.msdn.com/b/chuckw/archive/2012/04/26/xinput-and-windows-8-consumer-preview.aspx">XINPUT and Windows 8</a> and <a href="http://blogs.msdn.com/b/chuckw/archive/2012/04/02/xaudio2-and-windows-8-consumer-preview.aspx">XAudio2 and Windows 8</a> for guidance on handling this difference in Win32 desktop applications.</p>
<p><strong>Update:</strong> <a href="http://support.microsoft.com/kb/2670838">KB 2670838</a> and <a href="http://blogs.msdn.com/b/ie/archive/2013/02/26/ie10-for-windows-7-globally-available-for-consumers-and-businesses.aspx">Internet Explorer 10 for Windows 7</a> are now available. Users with the prerelease version installed should <a href="http://www.microsoft.com/en-us/download/details.aspx?id=36805">update</a> their systems (<a href="http://blogs.msdn.com/b/chuckw/archive/2013/02/26/directx-11-1-and-windows-7-update.aspx">DirectX 11.1 and Windows 7 Update</a>).</p>
<h1>Notes for users of the DirectX SDK</h1>
<p>The updated headers and link libraries needed to target the new components on Windows 8 and Windows 7 are in the Windows 8.x SDK as indicated in previous posts (see <a href="http://blogs.msdn.com/b/chuckw/archive/2012/03/22/where-is-the-directx-sdk.aspx">Where is the DirectX SDK?</a>). See <a href="http://msdn.microsoft.com/en-us/library/windows/desktop/ee663275.aspx">MSDN</a> for details on &lsquo;mixing&rsquo; the Windows 8.x SDK and legacy DirectX SDK if needed.</p>
<p>It is also important to note that the updated &ldquo;<a href="http://blogs.msdn.com/b/chuckw/archive/2012/11/30/direct3d-sdk-debug-layer-tricks.aspx">Debug Runtime</a>&rdquo; components in the Windows 8.x SDK are required on Windows 7 once KB 2670838 is installed. The legacy <em>DirectX SDK (June 2010)</em> &ldquo;Debug Runtime&rdquo; for Direct3D 10.x and Direct3D 11.0 is not compatible with Windows 10, Windows 8.x or Windows 7 once this update is applied. You can install the <a href="http://go.microsoft.com/fwlink/?LinkID=323507">Windows 8.x SDK standalone</a>, VS 2012 or VS 2013 which includes the Windows 8.x SDK, or the VS 2013 Remote Debugging Tools (<a href="http://go.microsoft.com/?linkid=9832075">x86</a> or <a href="http://go.microsoft.com/?linkid=9832085">x64</a>) to get the updated SDK Debug Layers files.</p>
<p>The legacy <em>PIX for Windows</em> tool in the <em>DirectX SDK (June 2010)</em> release does not support Direct3D 10.x or Direct3D 11.x applications on Windows 8, and after this update is applied it will no longer support these applications on Windows 7. Direct3D 9 application debugging continues to function.</p>
<p><strong>Related:</strong> <a href="http://blogs.msdn.com/b/chuckw/archive/2014/10/28/directx-sdk-tools-catalog.aspx">DirectX SDK Tools Catalog</a></p>
<h1>Notes for users of VS 2012</h1>
<p>Visual Studio 2012&rsquo;s <a href="http://msdn.microsoft.com/en-us/library/hh315751.aspx">Graphics Debugger</a> supports Direct3D 11.0 applications on Windows 7 and DirectX 11.x applications on Windows 8. Improved support for KB 2670838 is in <a href="http://blogs.msdn.com/b/chuckw/archive/2013/04/08/visual-studio-2012-update-2.aspx">VS 2012 Update 2</a>.</p>
<p>When using VS 2012 Update 1's new "<a href="http://blogs.msdn.com/b/chuckw/archive/2012/11/26/visual-studio-2012-update-1.aspx">v110_xp</a>" Platform Toolset the DirectX 11.1, WIC2, and related&nbsp;headers are not available.</p>
